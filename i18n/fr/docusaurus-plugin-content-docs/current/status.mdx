# État des fonctionnalités

Un aperçu de ce qui est disponible aujourd'hui par rapport à ce qui arrive.
avuru obs est en **pré-v0.1** et évolue vite — pour le plan, voir la
[Feuille de route](/docs/roadmap), et pour les changements livrés le
[Changelog](/changelog).

## Disponible maintenant

| Capacité | État | Depuis |
|---|---|---|
| Carte des services (arêtes d'appel **dérivées des traces**) | <StatusBadge status="shipped" label="Livré" /> | M1 |
| Traces — recherche, filtres (tags, ordre, durée, statut), heatmap de latence, aperçu RED par opération | <StatusBadge status="shipped" label="Livré" /> | M1 |
| Visualiseur de traces — chronologie, table des spans, flamegraph, statistiques, graphe, JSON | <StatusBadge status="shipped" label="Livré" /> | M4 |
| Comparaison de traces — diff structurel de deux traces | <StatusBadge status="shipped" label="Livré" /> | M4 |

:::info La carte des services aujourd'hui
Les arêtes de topologie sont **dérivées des spans de traces** (paires
Client/Server inter-services) — et avec le DaemonSet sensor, chaque service
HTTP/gRPC est tracé zéro-code : les applications non instrumentées apparaissent
aussi. Les **flux** L4 au niveau du noyau (pour les protocoles au-delà de ce que
l'instrumentation eBPF analyse) enrichiront encore la carte en **v0.2**.
:::

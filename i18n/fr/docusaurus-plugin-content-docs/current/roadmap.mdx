# Feuille de route

Où va avuru obs. Ceci est **indicatif, pas un engagement** — la portée et l'ordre
évoluent au fil de nos apprentissages. Le détail technique de référence vit dans
le [`ROADMAP.md`](https://github.com/avuruvision/avuru-obs/blob/main/ROADMAP.md)
du dépôt moteur ; cette page en est le résumé lisible. Pour ce qui fonctionne
aujourd'hui, voir l'[État des fonctionnalités](/docs/status) ; pour ce qui a été
livré, le [Changelog](/changelog).

## Étoile polaire

:::tip Le coin
Un cluster Kubernetes neuf → **carte des services en direct en moins de cinq
minutes**, zéro changement applicatif. Chaque jalon est jugé à cette aune, et
c'est vérifié par une porte d'intégration continue.
:::

## v0.1 — le coin (première version taguée)

Les niveaux de signaux livrés pour `0.1` :

| Niveau | Signal | État |
|---|---|---|
| **Complet** | Carte des services + métriques RED ; explorateur de traces (waterfall, recherche) | <StatusBadge status="shipped" label="Livré" /> explorateur · <StatusBadge status="in-progress" label="En cours" /> UI RED |
| **Basique** | Logs (collecte, recherche plein texte, corrélation `trace_id`) | <StatusBadge status="shipped" label="Livré" /> |
| **Léger** | Profilage continu (flame graphs CPU par service) | <StatusBadge status="planned" label="Prévu" /> |
| **Support** | Métriques d'infra (CPU/mémoire/réseau des nœuds/pods) | <StatusBadge status="planned" label="Prévu" /> |

Plus la promesse forte : un **remplaçant OTLP « drop-in »** — les applications
déjà instrumentées migrent en changeant seulement l'endpoint d'export, sans
changement de SDK ni de code.

## Au-delà de v0.1 (indicatif)

- **v0.2 — auth :** OIDC derrière l'interface de fournisseur d'auth du `hub`
  (v0.1 livre un mot de passe admin local). La jointure entreprise — fournisseur
  d'auth, colonne `tenant`, objets de politique de rétention — est intégrée dès
  v0.1, donc cela arrive sans réécriture.
- **Plus de clients :** l'API du hub est le contrat agnostique ; la SPA n'en est
  qu'un client léger. Une source de données **Grafana** et une **CLI** sont
  prévues.
- **Réévaluation du stockage :** ClickHouse reste derrière l'interface
  `storage.Store` ; GreptimeDB sera réévalué mi-2027 sans changer le code du hub.

## Comment cette feuille de route change

Ouvrez une issue ou une discussion pour proposer un changement de cap ; les
éléments plus importants passent par une Avuru Enhancement Proposal avant
implémentation. Les modifications passent par une pull request classique.

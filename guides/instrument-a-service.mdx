# Instrument a service

The eBPF `sensor` gives you the service map for free. Add an OTLP SDK where you
want **in-process detail** — database statements, business attributes, custom
spans.

1. Add the OpenTelemetry SDK / auto-instrumentation for your language (see
   [Integrations](/integrations)).
2. Point the exporter at the gateway:

   ```bash
   export OTEL_EXPORTER_OTLP_ENDPOINT=http://avuruops-gateway:4318
   ```

3. Redeploy. Your spans now nest inside the eBPF-discovered traces.

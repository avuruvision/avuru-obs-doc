# Feature status

A snapshot of what's live today versus what's coming. avuru obs is **pre-v0.1**
and moving fast — for the plan see the [Roadmap](/docs/roadmap), and for shipped
changes the [Changelog](/changelog).

## Available now

| Capability | Status | Since |
|---|---|---|
| Service map (call edges **derived from traces**) | <StatusBadge status="shipped" /> | M1 |
| Traces — search, filters (tags, order, duration, status), latency heatmap, per-operation RED overview | <StatusBadge status="shipped" /> | M1 |
| Trace viewer — timeline, spans table, flamegraph, statistics, trace graph, JSON | <StatusBadge status="shipped" /> | M4 |
| Trace comparison — structural diff of two traces | <StatusBadge status="shipped" /> | M4 |
| Logs — full-text search, severity/service filters, `trace_id` correlation | <StatusBadge status="shipped" /> | M1 |
| System Status — component health, storage, retention, disk | <StatusBadge status="shipped" /> | M1 |
| Sensor DaemonSet — **zero-code** eBPF traces + RED (OBI), **zero-config** log collection | <StatusBadge status="shipped" /> | M2 |
| Service inventory — sortable RED table, drill-down to traces | <StatusBadge status="shipped" /> | M2 |
| RED metrics dashboard — rate/errors/duration charts per service | <StatusBadge status="shipped" /> | M3 |
| Node / pod health — CPU, memory, network per node & pod (kubeletstats) | <StatusBadge status="shipped" /> | M3 |
| Continuous profiling — always-on CPU flame graphs per service | <StatusBadge status="shipped" /> | M4 |

## Coming

| Capability | Status | Target |
|---|---|---|
| eBPF **flow** topology enriching the service map (custom L4 tracer) | <StatusBadge status="planned" /> | v0.2 |
| Configuration / OpAMP controls in the UI | <StatusBadge status="planned" /> | v0.2 |
| Auth / multi-tenant (seam already in place) | <StatusBadge status="planned" /> | v0.2 |

:::info Service map today
Topology edges are **derived from trace spans** (cross-service Client/Server
pairs) — and with the sensor DaemonSet, every HTTP/gRPC service is traced
zero-code, so uninstrumented apps appear too. Kernel-level L4 **flows** (for
protocols beyond what eBPF instrumentation parses) will further enrich the map
in **v0.2**.
:::

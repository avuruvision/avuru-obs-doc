# eBPF mode (zero-code)

The **`sensor`** is an eBPF DaemonSet that attaches to kernel probes
(`tcp_connect`, `tcp_close`, …) to reconstruct your service map and L4 flows —
**without touching application code**.

## What you get for free

- Service map (who talks to whom)
- RED metrics (Rate, Errors, Duration) per service
- L4 flow records

## Requirements

- Linux kernel **≥ 5.8 with BTF** for full functionality.
- The DaemonSet runs with the capabilities needed to load eBPF programs (see
  [Security](/docs/operations/security)).

## When to add OTLP

eBPF can't see *inside* a process. For DB statements, business attributes and
custom spans, add an OTLP SDK and use the [OTLP bridge](/docs/setup/otel-bridge).
eBPF and OTLP spans share the same storage and service identity.

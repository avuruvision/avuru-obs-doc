# Profiling

Continuous profiling samples CPU (and, over time, heap) and attributes the cost
back to your code — so you can answer "which function burned the time in this
slow span?".

- **Source** — the OpenTelemetry eBPF profiler in the `sensor` pod.
- **Correlation** — by resource attributes and time window with traces.

:::info Planned — M4
Continuous profiling is **not yet available** in the UI. The OpenTelemetry eBPF
profiler runs in the `sensor`, but the storage schema and flame-graph UI land in
**M4**, isolated behind an adapter and expected to change. See the
[Roadmap](/docs/roadmap) and [Feature status](/docs/status).
:::
